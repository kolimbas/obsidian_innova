# Research — 08 Mayo 2026

---

## 🔋 Tokens

**Context Mode (Claude Code)** — Compresión nativa 63x en sesiones largas. Reemplaza el contexto acumulado por una versión comprimida. Activarlo en proyectos grandes de clientes, especialmente en fase de integración Supabase.
`https://claudefa.st/blog/guide/changelog`

**Patrón raw/wiki (Karpathy)** — Convertir archivos, transcripts y notas en una wiki comprimida. Hasta 95% de reducción de tokens. Aplicar a los docs de cada cliente antes de pasarlos a Claude Code.
`https://buildtolaunch.substack.com/p/claude-code-token-optimization`

---

## 🌐 Desarrollo Web

**Vercel + Supabase First-Party** — Supabase ahora es integración oficial en Vercel. Las credenciales se inyectan automáticamente en env vars. En fase de deploy ya no se copian y pegan manualmente.
`https://vercel.com/marketplace/supabase`

---

## 🤖 Claude Code

**`/recap`** — Nuevo comando que resume todo lo hecho en la sesión. Usar al cerrar cada sesión de cliente para registrar el estado. Input ideal para el CONTEXT.md del proyecto.
`https://code.claude.com/docs/en/changelog`

---

## ⚡ Automatización

**n8n HITL** — Aprobación humana explícita antes de que el agente ejecute herramientas. Configurar en el bot de WhatsApp antes de enviar propuestas de precio o cerrar tratos automáticamente.
`https://docs.n8n.io/release-notes/`

---

> [!summary] Top 3 acciones
> 1. Activar Context Mode en Claude Code → 63x, gratis, nativo
> 2. Conectar Supabase desde Vercel Marketplace → ahorra 5-10 min por deploy
> 3. Agregar HITL al bot de WhatsApp en n8n → control sobre cierres automáticos
